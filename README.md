# 🔥 FOMO Index

**صفحة واحدة، بدون أي تدخل من المستخدم — تعرض مؤشراً يقيس درجة الفومو الحقيقية في سوق العملات الرقمية من خلال حركة الأموال الفعلية (Money Flow)، بأسلوب مشابه لـ Fear & Greed Index.**

---

## 1) الفكرة الأساسية

`FOMO Index` **ليس** نسخة من Fear & Greed Index، **وليس** مؤشر مشاعر، **وليس** مؤشر تحليل فني.

الفلسفة الوحيدة للمؤشر:

> هل تدخل أموال جديدة إلى السوق فعلياً (عبر منصات التداول) بسبب الفومو، أم لا؟

المؤشر لا يستخدم السعر، ولا المشاعر، ولا الأخبار في حساب قيمته. السعر يُعرض فقط بجانب القيمة للسياق، دون أن يدخل في المعادلة.

---

## 2) تجربة الاستخدام (مطابقة لـ Fear & Greed Index)

عند فتح الصفحة:
1. تبدأ عملية جلب البيانات وحساب المؤشر **تلقائياً** فور التحميل.
2. تظهر النتيجة مباشرة: عداد Gauge كبير + القيمة + الحالة (`Very Low` / `Low` / `Neutral` / `High` / `Extreme FOMO`) + تفسير قصير + سعر BTC + تاريخ آخر تحديث + رسم بياني لآخر 30 يوماً.
3. **لا يوجد أي عنصر تحكم**: لا أزرار، لا قوائم اختيار مدى البيانات، لا تحميل يدوي، لا صفحات إضافية، لا وضع ليلي/نهاري قابل للتبديل.
4. الصفحة تُحدَّث تلقائياً كل ساعة (عبر إعادة تحميل ذاتية في المتصفح)، والبيانات تُخزَّن مؤقتاً (Cache) لمدة ساعة فلا يُعاد جلبها في كل تحميل.

---

## 3) الموثوقية — لا يتوقف التطبيق أبداً

هذه هي أهم نقطة في التصميم الحالي. مبدأ الملف `app.py` بالكامل:

> أي خطأ (فشل شبكة، بيانات ناقصة، استثناء غير متوقع) يُعالَج محلياً بـ `try/except` ولا يُسمح له أبداً بإيقاف التطبيق أو إظهار شاشة عطل (Traceback) للمستخدم.

### سلسلة التدهور التدريجي (Graceful Degradation)

```
1) محاولة حساب المؤشر من البيانات الحية (CoinGecko + DeFiLlama + data_providers)
        │  فشل أي مصدر → يُجرَّب المصدر الاحتياطي التالي تلقائياً (بلا توقف)
        ▼
2) إذا فشلت كل المصادر الحية بالكامل
        │
        ▼
3) تحميل "آخر قراءة سليمة محفوظة" من القرص (persistence.py)
        │  تظهر مع تنبيه صغير: "⚠️ يتم عرض آخر قراءة سليمة محفوظة"
        ▼
4) إذا لم توجد أي قراءة محفوظة من الأساس (أول تشغيل + فشل كامل)
        │
        ▼
   رسالة واحدة فقط: "البيانات غير متوفرة حالياً" — بدون أي خطأ فني معروض للمستخدم
```

### آليات الحماية المطبّقة في كل طلب شبكة (`data_providers/http_utils.py`)

| الآلية | القيمة |
|---|---|
| Timeout لكل طلب | 20 ثانية |
| إعادة المحاولة (Retry) | 3 محاولات مع تراجع أسي (Exponential Backoff) قبل الانتقال للمصدر الاحتياطي |
| Cache | `st.cache_data(ttl=3600)` — ساعة كاملة، لا يُعاد الجلب عند كل Refresh |
| معالجة الاستثناءات | كل استدعاء API محاط بـ `try/except Exception` عام — لا يوجد استثناء غير مُعالَج في كامل المشروع |
| DataFrame فارغ/ناقص | كل دالة تتحقق من `df.empty` / `dropna` قبل الاستخدام، وتُرجع بيانات احتياطية بدل الانهيار |

هذا التصميم يمنع تحديداً كل الأعطال المذكورة: `Build Failed`, `Runtime Error`, `Bad Gateway`, `Module Error`, `API Error`, `KeyError`, `IndexError`, `ValueError`, `ConnectionError`.

## 4) مصادر البيانات وطبقة المزودين القابلة للتوسعة

المشروع يعمل **100% مجاناً بدون أي مفتاح API** بشكل افتراضي، عبر طبقة `data_providers/` القابلة للتوسعة (راجع تفاصيلها وتبرير كل وزن في تعليقات `config.py` و`data_providers/*.py`).

| المكوّن | الوزن | المصدر الافتراضي (مجاني) | أفضل بديل مدفوع |
|---|---|---|---|
| **Stablecoin Exchange Inflows** | 50% | CoinGecko (تقدير من نشاط تداول USDT/USDC) أو Whale Alert (بمفتاح مجاني) | CryptoQuant / Glassnode / Nansen / Arkham |
| **BTC Exchange Netflow** | 25% | CoinGecko (تقدير من حجم التداول والسعر) | CryptoQuant / Glassnode |
| **BTC Exchange Balance** | 15% | CoinGecko (نفس التقدير أعلاه، تراكمي) | CryptoQuant / Glassnode |
| **Stablecoin Supply** (سياق فقط) | 10% | DeFiLlama (معروض عالمي حقيقي، مجاني بالكامل) | — |

الاستبدال بمزود مدفوع أدق يتم بسطر واحد فقط في `config.py` (وضع المفتاح)، دون أي تعديل على `fomo_index.py` أو `app.py`.

## 5) هيكل المشروع (بعد التنظيف)

```
Fomo-Index/
├── app.py                       # نقطة التشغيل الوحيدة — صفحة واحدة فقط
├── config.py                     # كل الأوزان، النوافذ الزمنية، الثوابت
├── fomo_index.py                 # منطق حساب المؤشر (دوال نقية وقابلة للاختبار)
├── data_sources.py               # تنسيق مصادر البيانات (orchestration فوق data_providers)
├── data_providers/                # طبقة مزودين قابلة للتوسعة (مجاني <-> مدفوع)
│   ├── base.py                   # الواجهة الموحدة لكل مزود
│   ├── http_utils.py              # طلبات HTTP (timeout=20s + retry + fallback)
│   ├── stablecoin_inflows.py      # مزودو Stablecoin Exchange Inflows
│   └── btc_exchange_flows.py      # مزودو BTC Exchange Netflow/Balance
├── persistence.py                 # حفظ/استرجاع آخر قراءة سليمة (خط الدفاع الأخير)
├── backtest.py                    # أداة اختبار تاريخي (تُشغَّل يدوياً، غير مستخدمة من app.py)
├── requirements.txt
├── runtime.txt                    # إصدار Python المستخدم على Render (3.12.3)
├── render.yaml                    # إعداد نشر جاهز لـ Render
├── .streamlit/config.toml         # مظهر داكن ثابت + إعدادات خادم
└── README.md
```

`app.py` هو **نقطة التشغيل الوحيدة**؛ كل الملفات الأخرى وحدات مستوردة (modules) وليست صفحات أو نقاط دخول مستقلة. `backtest.py` أداة تطوير اختيارية تُشغَّل يدوياً بـ `python backtest.py` ولا علاقة لها بتشغيل الموقع نفسه.

## 7) النشر على Render (Web Service)

**الطريقة الأسهل — Blueprint (ملف `render.yaml` جاهز):**
1. ارفع المشروع كاملاً إلى GitHub بدون أي تعديل.
2. من لوحة Render اختر **New → Blueprint**، واربطه بالمستودع.
3. Render سيقرأ `render.yaml` تلقائياً ويُنشئ الخدمة بالإعدادات الصحيحة (لا حاجة لأي إدخال يدوي).

**الطريقة اليدوية (New Web Service):**
| الإعداد | القيمة |
|---|---|
| Environment | Python 3 |
| Build Command | `pip install -r requirements.txt` |
| Start Command | `streamlit run app.py --server.port $PORT --server.address 0.0.0.0 --server.headless true` |

لا يحتاج المشروع أي متغيرات بيئة إلزامية للعمل بالوضع المجاني الافتراضي.

## 9) طريقة التطوير والتوسعة

- **تغيير الأوزان أو النوافذ الزمنية:** عدّل `config.py` فقط.
- **إضافة مزود بيانات جديد (مجاني أو مدفوع):** أنشئ صنفاً جديداً يرث من `data_providers.base.DataProvider` وأضفه لقائمة المرشحين في `get_stablecoin_inflow_providers()` أو `get_btc_exchange_flow_providers()`.
- **تغيير مدة الكاش أو التحديث التلقائي:** `config.CACHE_TTL_SECONDS` و`config.AUTO_REFRESH_SECONDS`.
- **اختبار منطق الحساب بمعزل عن الواجهة:** استورد `fomo_index.py` مباشرة (دوال نقية بدون أي اعتماد على Streamlit).
- **الاختبار التاريخي:** `python backtest.py` (أداة منفصلة، غير مستخدمة من `app.py`).

---

## 10) القيود الحالية

- بيانات Stablecoin Exchange Inflows و BTC Exchange Netflow/Balance تقديرية (Proxy) في الوضع المجاني الافتراضي، حتى تتم إضافة مفتاح مزود حقيقي.
- طبقة `persistence.py` تحفظ على القرص المحلي للخدمة (تخزين مؤقت خلال عمر الحاوية)، وليست قاعدة بيانات دائمة عبر عمليات إعادة النشر (Redeploys) على Render.
- المؤشر أداة تحليلية/تعليمية وليس توصية استثمارية.
